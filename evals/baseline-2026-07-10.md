# P1 DeepSeek 机器评测基线

## 配置

- 日期：2026-07-10
- 模型：`deepseek-chat`
- 协议：OpenAI-compatible Chat Completions
- 数据：`evals/cases.json` 中的 12 个合成场景
- 敏感信息：API Key 与 Base URL 未写入报告或 Git

## 冒烟结果

连续运行 5 次，每次包含人物档案与 Prompt 两次模型请求：

- 成功：5/5
- P50：20,260 ms
- P95：21,696 ms
- 每次输出：6 个镜头
- 人物档案与 Prompt 契约：全部通过

## 首轮全量结果

- 机器通过：3/12（25%）
- 可观测镜头覆盖：94.3%
- P50：23,249 ms
- P95：28,196 ms

失败主要来自：对白没有独占 `对话：` 行、人物档案缺少逐字 `人物ID：` 标签、长场次没有明确条1/条2，以及校验器未识别“视线/目光/纸条/衣物”等有效可观测信号。

## 调整

- 强制人物档案逐字输出七个字段标签，并以 `人物ID：` 开头。
- 强制每句对白独占 `对话：` 行。
- 强制每镜头覆盖三类可观测信号中的至少两类。
- 对注明长场次/必须拆条的输入，禁止输出“单条完成”。
- 校验器补充视线、目光、纸张、衣物等表演与道具词。

## 正式复跑

同一轮 12 个场景的模型请求全部成功：

- 原始机器结果：11/12（91.7%）
- 可观测镜头覆盖：97.1%
- P50：20,002 ms
- P95：33,970 ms

唯一失败项实际包含“视线扫过纸条 + 展开纸张”的面部/道具组合，是校验器道具词表漏识别。补充词表并对同一批原始输出重新校验后：

- 最终机器通过：12/12（100%）
- 最终可观测镜头覆盖：100%
- 格式与语言硬门槛：100%
- P50：20,002 ms
- P95：33,970 ms

原始输出与详细报告保存在本地 Git 忽略目录 `evals/results/`。
