# 结构化与物理因果提示设计

## 目标

把截图中的有效经验转成当前 `performance-prompter` 可复用、可验证的生成规则，提高 Seedance 文戏提示词的动作清晰度、环境真实感和稳定性，同时不牺牲现有表演密度与 B 格式契约。

## 采纳原则

1. 每个镜头先锚定主体，再描述主体的动作与状态；环境、镜头和氛围放在主体之后，避免信息乱序。
2. 动作优先按“整体姿态/重心 -> 头部 -> 眼神与面部 -> 手部与道具 -> 细节反馈”组织。它是默认写作顺序，不是要求每个镜头机械填满所有层级。
3. 出现风、雨、水、光、雾、接触或材质时，写出可见的原因与结果，例如“雨点击中伞面 -> 水花溅起”，而不是只写“在雨中”。不得凭空添加与场景无关的物理效果。
4. 保留负向约束，并扩展到面部变形、肢体错乱、手指异常、闪烁、过曝/过暗和低画质模糊。截图中的效果百分比没有可复核实验，不写入 Skill 结论。

## 落点

- `SKILL.md` 只保留核心工作流和引用入口。
- 新增 `references/structured-physical-direction.md` 承载详细顺序、物理因果模板和反例。
- `server.js` 将该参考资料载入模型上下文，并把关键要求写入生成硬约束。
- `scripts/lib/prompt-contract.mjs` 更新固定负向段，并对明确出现的风、雨、雾、阳光/逆光场景校验物理反馈。
- 单元测试覆盖参考资料接线、旧负向模板失效、物理因果通过/失败两条路径；固定 12 场景评测用于观察回归。

## 非目标

- 不修改网页 UI、公共 API、数据格式或模型配置。
- 不宣称未经项目 A/B 测试验证的成功率提升。
- 不把摄影风格扩展为复杂运镜系统；每个镜头仍以人物表演为主。
